## 6. Workflow publikacji (per post) — REALNY dla tego repo (statyczny HTML)

> ⚠️ To NIE jest projekt Next.js. Nie ma `/src/blog/` ani renderowania MDX. MDX = tylko szkic treści. Stroną na żywo jest samodzielny plik `.html` w **roocie** repo, zbudowany ręcznie wg szablonu istniejących 12 wpisów.

1. [ ] Szkic treści jako `.mdx` w `Wpisy-blogowe/` (wg zasad z sekcji 4) — dostarcza Michał
2. [ ] Konwersja MDX → samodzielny `[slug].html` w **roocie** repo, skopiowany ze struktury istniejącego wpisu (np. `ile-kosztuje-uszycie-partii-odziezy.html`)
3. [ ] `<head>`: title ≤60 zn., meta description 150–160 zn., `canonical = https://szwalnia-isabell.pl/[slug].html`, OG, GA4, `/style.css`
4. [ ] Schema JSON-LD: `BlogPosting` + `FAQPage` + `BreadcrumbList` (+ `Organization`). `datePublished` = **realna data publikacji** (NIE przyszła)
5. [ ] Linkowanie wewnętrzne: min. 2 linki do podstron usług + cross-linki wg mapy kanibalizacji (sekcja 5)
6. [ ] Dopisać URL do `sitemap.xml` (z `lastmod`) i dodać kafelek wpisu do `blog.html`
7. [ ] Grafiki — **dopiero na końcu** (placeholder do tego czasu); zdjęcia webp + `srcset`, alt z frazą
8. [ ] `git add [konkretne pliki] && git commit && git push` → Vercel auto-deploy (30–60 s)
9. [ ] Google Search Console → URL Inspection → Request Indexing
10. [ ] (opcjonalnie) Udostępnić na social media + post na Google Business Profile
